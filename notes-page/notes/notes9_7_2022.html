  • Git Repository : tracks and saves the history of all changes made to the files in a Git project.
  • local : physical, locally-managed repositories to store internal binaries.
  • remote : versions of project hosted on the internet or network
  • gitkraken.com
  • Main Deployment :
  • Feature Branch : upstream branch
  • git commit -m : command used to save changes to the local repository. Must state which changes to include.
  • git add : the first command in a chain of operations that directs Git to save a snapshot of the current project state into commit history. On its own will promote pending changes from the working directory to the staging area.
  • git status : displays the state of the working directory and the staging area. Let's you see what changes have been staged, which haven't, and which aren't being tracked.
  • repot :
  • Intitial Commit : the first commit to start a new project.
  • Technical Documentation :
  • Staging for Commit : to prepare for a commit.
  • Conventions :
  • git log : a utility tool to review and read a history of everything that happens to a repository. A record of commits.
  • git help : a manual page display all information about git.
  • .gitignore : tells Git which files to ignore when committing to the repository.
  • git init : command used to initialize a new git repository or reinitialize a existing one. Transforms current directory into Git repository. Creates a new master branch.
  • git push : command to upload local repository content to a remote repository. Transfer commits from local repository to remote repository. Counterpart to git fetch.
  • git status : command displays the state of the working directory and the staging area.
